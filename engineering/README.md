# Engineering Skills

Daily code work, issue management, planning, debugging, testing, and architecture.

- [create-issues](./create-issues/SKILL.md) - Scan a repository for validated code quality findings and create one GitHub issue per unique finding.
- [diagnose](./diagnose/SKILL.md) - Run a disciplined diagnosis loop for hard bugs and performance regressions.
- [fix-issue](./fix-issue/SKILL.md) - Resolve a GitHub issue with a confirmed plan, minimal fix, validation, and PR.
- [grill-with-docs](./grill-with-docs/SKILL.md) - Stress-test a plan against project language and documented decisions.
- [improve-codebase-architecture](./improve-codebase-architecture/SKILL.md) - Find architecture improvement opportunities using domain language and ADR context.
- [improve-issue](./improve-issue/SKILL.md) - Improve or decompose an existing GitHub issue into actionable work.
- [prototype](./prototype/SKILL.md) - Build a throwaway prototype to test a design before committing to it.
- [setup-skills](./setup-skills/SKILL.md) - Set up per-repo configuration consumed by the engineering skills.
- [tdd](./tdd/SKILL.md) - Build features or fixes using a red-green-refactor loop.
- [to-issues](./to-issues/SKILL.md) - Break a plan, spec, or PRD into independently grabbable issues.
- [to-prd](./to-prd/SKILL.md) - Turn current conversation context into a PRD.
- [triage](./triage/SKILL.md) - Triage issues through a role-driven state machine.
- [zoom-out](./zoom-out/SKILL.md) - Explain unfamiliar code from a higher-level system perspective.
